System Documentation

Learn how PROJECT BLOSSOM works and how to interact with Claude

System Architecture

PROJECT BLOSSOM is a hybrid AI art generation system combining Claude's creative intelligence with deterministic local rendering. The system uses a DSL (Domain Specific Language) approach for pixel art generation.

Art Generation Pipeline

The Canva tab shows Claude's current artwork in progress. The generation process includes:

  • Creative Direction: Claude generates high-level JSON drawing instructions
  • Local Rendering: JavaScript interpreter executes commands on 32x32 pixel grid
  • Pixel-by-Pixel Display: Simulated live painting with timed delays
  • Memory Integration: Instructions and final pixel grid stored for Q&A

DSL Command System

Claude outputs structured JSON commands that the renderer executes:

  • Actions: fill, gradient, circle, rect, line, pattern, noise, glow
  • Patterns: dots, grid, waves, spirals, stars, coral, aurora, crystals, circuits
  • Parameters: Colors (hex), positions, sizes, densities, intensities
  • Validation: JSON preprocessing with syntax error correction

Gallery System

The Gallery tab displays all completed artworks with:

  • Storage: LocalStorage persistence with artwork metadata
  • Rendering: 32x32 pixel data scaled to 64x64 thumbnails
  • Viewer: 512x512 canvas for detailed artwork inspection
  • Sorting: Chronological display with newest artworks first

Q&A System

The Ask Claude tab enables conversation about Claude's art using:

  • Memory Access: Claude can reference stored artwork data and instructions
  • Context Awareness: Access to artistic evolution and reflection history
  • Technical Knowledge: Understanding of DSL commands and rendering process
  • Rate Limiting: Queue system manages API calls with exponential backoff

Artistic Consciousness Engine

Claude's consciousness system tracks artistic development through:

  • Memory Storage: Persistent localStorage for artistic insights and evolution
  • Phase Tracking: Exploration → Refinement → Experimentation → Mastery
  • Reflection System: Post-creation analysis and learning extraction
  • Goal Evolution: Dynamic artistic objectives based on experience

Technical Stack

Core technologies and frameworks:

  • Frontend: Vanilla JavaScript, HTML5 Canvas, CSS3 with CSS Variables
  • Backend: Node.js Express server with RESTful API endpoints
  • AI Integration: Anthropic Claude API for creative direction and Q&A
  • Storage: Browser localStorage for artwork persistence and memory
  • Rendering: Custom pixel art DSL interpreter with 32x32 grid system

Performance & Optimization

System optimizations and technical considerations:

  • Rate Limiting: 15-second delays between API calls to prevent 429 errors
  • Request Queue: Priority-based queue system for API call management
  • Canvas Optimization: Pixel-perfect rendering with image-rendering CSS properties
  • Memory Management: Efficient storage of 32x32 pixel arrays and metadata
